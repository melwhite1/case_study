Objective

The objective was to create an app for users to view and search recipes of varying difficulty, ingredients and cooking time. Users can compare recipes by viewing a variety of charts and graphs.

Role

My role in this project was to build the application using Django for my web development course. I set up the models within the framework, filled the backend data, displayed data for the front-end, and enabled users to preform actions within the site.

Tech Stack

About

Recipebook is a web app, developed using Python and Django, that provides users with access to recipes. Users are able to create an account, view a list of recipes, and sort/display recipes on a variety of graphs and charts. Recipebook was a personal project I built as part of my web development course at CareerFoundry to deomnstrate my mastery of fullstack development.

Approach

Server-Side

I initiated the server-side using Django to create modules for my application. The modules included information about individual recipes such as name, ingredients, cooking time (in minutes), difficulty and a picture. After making migrations and running my server, I used the Django admin page to populate the Recipes list. To test the modules functions I implemented a testing module with sample data.

View on Github

Client-Side

After creating the Django database, I started to build the interface users would need to login, view recipes, and search for specific criteria. It is a single-page, responsive application devloped using Django and served on Heroku. It provides several interface views, including, but not limited to, a main view (shows a list of all recipes), single recipe view (shows data about a single recipe), a login view, a home page and a search page (where users can search recipes by various criteria).
